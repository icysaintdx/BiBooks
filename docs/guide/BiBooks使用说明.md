## 1. 软件概述与界面布局

BiBooks 是一款基于 Electron + React 的桌面级智能标书工具，核心能力是把"招标文件"转换为"可投标的技术方案 Word 文档"，并提供查重、废标项检查、知识库、报价等配套能力。

### 1.1 整体界面结构

- **左侧主菜单**：纵向排列全部功能模块入口，点击切换主工作区。
- **中部工作区**：当前模块的主要操作区域，技术方案模块按步骤（STEP）线性推进。
- **浮动工具条（FloatingToolbar）**：固定悬浮于工作区，提供"上一步 / 下一步 / 导出 / 重置"以及版本、竞品、合规、协同等快捷入口。
- **全局提示（Toast）**：所有操作的成功 / 失败 / 提示信息会以浮层方式在界面提示。

### 1.2 主菜单模块一览

| 菜单 | 说明 | 状态 |
| --- | --- | --- |
| 技术方案 | 从招标文件到技术方案正文的 6 步流水线 | 可用 |
| 商务标 | 报价、条款、资质等商务文件编制 | 开发中（Demo 预览） |
| 报价管理 | 报价方案与价格策略管理 | 可用 |
| 知识库 | 行业 / 通用参考文档管理与目录对齐 | 可用 |
| 企业知识库 | 企业自有资料（资质、业绩、模板）管理 | 可用 |
| 标书查重 | 多份标书重复度检测（4 维度） | 可用 |
| 废标项检查 | 废标风险 / 错别字 / 逻辑谬误检查 | 可用 |
| 投标机会 | 招标公告聚合与投标决策 | 开发中（Demo 预览） |
| 设置 | 模型、解析器、API 服务等配置 | 可用 |

> 标注"开发中"的模块界面会显示"正在开发中，敬请期待"遮罩，按钮处于禁用状态，仅供效果预览。

## 2. 首次使用配置（必读）

首次打开软件，需先到「设置」完成模型配置，否则技术方案、解析、生成等 AI 能力无法使用。

### 2.1 配置文本模型（必填）

进入「设置 → 文本模型」，选择一个提供商并填写凭证：

- **在线提供商**：金龙中转站（推荐）、火山方舟、小米 token plan、DeepSeek、龙猫，以及"自定义"。
- **本地离线提供商**：Ollama、LM Studio、llama.cpp、vLLM，适合本地部署模型、无需联网的场景。

填写要点：
- `API Key`：在线提供商需要填入对应平台申请的密钥；本地提供商通常留空。
- `Base URL`：选择提供商后会自动填入默认地址，可按需修改（如本地端口）。
- `模型名称`：填入实际可用的模型 ID。
- 填写后点击「测试连接」验证可用性，再「保存」。

### 2.2 配置生图模型（可选）

进入「设置 → 生图模型」。若需要在正文中自动插入 AI 配图，需在此配置生图模型并测试为"可用"状态。
- 状态分为：未测试 / 可用 / 不可用。
- 仅当状态为"可用"时，正文生成配置里的"使用 AI 生图"开关才可启用。
- 不配置也能正常编写标书，只是无法自动配图（仍可使用 Mermaid 图）。

### 2.3 配置文件解析器（重要）

进入「设置 → 文件解析」。招标文件 / 投标文件解析依赖此配置。
- 部分文档格式（如老版 .doc）需要本地安装 LibreOffice 才能解析，若缺失会提示安装。

### 2.4 其他设置项

- **通用**：开发者模式开关、软件更新检查等。开启开发者模式后，正文生成页会显示配图统计等调试信息。
- **API 服务**：本地 API 服务的启停与状态查看。
- **关于**：版本号、更新检查。

> 配置完成后即可返回「技术方案」开始正式编写。

## 3. 技术方案模块（核心功能）

技术方案是软件的核心，按 6 个步骤线性推进。顶部步骤标签依次为：上传招标文件 → 招标文件解析 → 目录生成 → 全局事实设定 → 生成正文 → 扩写改写。每一步完成后才能进入下一步，浮动工具条的"下一步"按钮会给出未完成原因提示。

关键特性：**所有 AI 任务在后台运行，切换页面或切到其他模块都不会中断**；任务进度通过事件实时回传到界面。

### 3.1 STEP 01 上传招标文件

- **作用**：导入招标文件（Word/PDF 等），软件自动解析为 Markdown 文本作为后续所有步骤的输入。
- **操作**：
  1. 在上传区选择本地招标文件。
  2. 软件调用文件解析器，将文件转为 Markdown 并在页面右侧预览。
  3. 解析完成后，"下一步"按钮解锁。
- **效果**：解析后的 Markdown 全文会被保存，可在本步随时查看。
- **注意**：未上传招标文件时无法进入下一步，提示"上传完招标文件后才能进入下一步"。

### 3.2 STEP 02 招标文件解析

- **作用**：基于招标文件 Markdown，由 AI 抽取关键信息，拆分为多个解析任务（如项目概况、技术要求、评分办法等）。
- **解析模式**：支持"关键项（key）"等模式，按需抽取必需的解析任务。
- **操作**：
  1. 选择解析模式。
  2. 启动解析，多个解析子任务并发执行，页面显示整体进度条与各子任务状态。
  3. 必需的解析任务全部成功并有内容后，"下一步"解锁。
- **产出**：
  - **项目概况（projectOverview）**：用于后续目录生成和正文。
  - **技术要求（techRequirements）**：技术方案的核心依据。
  - 其他结构化解析结果（项目信息等）供竞品分析、合规检查复用。
- **可选**：本步可启动「评分分析」（见第 4 章），抽取评分办法供"按评分项对齐"的目录生成使用。
- **注意**：解析未完成时提示"招标文件解析完成后才能进入目录生成"。

### 3.3 STEP 03 目录生成

- **作用**：根据项目概况与技术要求，生成技术方案的章节目录（树状结构）。
- **目录模式**：
  - **自由生成（free）**：由 AI 根据项目内容自主组织章节结构。
  - **按评分项对齐（aligned）**：让目录章节与招标评分项一一对应，最大化评分覆盖（推荐用于评分制项目）。
- **参考知识库**：可勾选知识库中的参考文档（referenceKnowledgeDocumentIds），让目录生成参考行业模板与既有资料。
- **操作**：
  1. 选择目录模式（自由 / 按评分项对齐）。
  2. （可选）选择参考的知识库文档。
  3. 启动目录生成，后台任务运行，完成后展示树状目录。
  4. 目录支持手动编辑：增删章节、调整层级、重命名，系统会自动重新编号（如 1 / 1.1 / 1.1.1）。
- **效果**：生成的目录会清空旧正文内容，作为正文生成的骨架。修改目录会联动重置后续的全局事实和正文（避免不一致）。
- **注意**：目录未生成时提示"目录生成完成后才能进入全局事实设定"。

### 3.4 STEP 04 全局事实设定

- **作用**：设定整篇标书必须保持一致的"全局事实"（如公司名称、项目名称、关键数据、技术参数、人员配置等），避免各章节由 AI 分别生成时出现前后矛盾。
- **操作**：
  1. 进入本步后，可启动 AI 自动从招标文件与解析结果中提取全局事实，按分组（GlobalFactGroup）呈现。
  2. 逐条核对、修改、补充全局事实，确保准确（这是后续正文一致性的基准）。
  3. 保存全局事实，任务状态变为成功后"下一步"解锁。
- **效果**：全局事实会在正文生成时注入到每个小节的生成上下文，并在"全文一致性审计"阶段用作冲突比对的依据。
- **注意**：
  - 全局事实为空或任务未成功时提示"全局事实设定完成后才能进入正文生成"。
  - 修改全局事实会联动重置已生成的正文（保证一致性）。

### 3.5 STEP 05 生成正文（核心步骤）

正文生成按目录的**叶子小节**并发生成，是整个流程中功能最丰富的一步。

#### 3.5.1 页面布局

- **顶部命令栏**：显示小节总数、已生成数、总字数三项统计；右侧为"生成配置"齿轮按钮和主操作按钮。
- **左侧目录面板**：树状展示全部章节，每个节点显示状态（待生成 / 生成中 / 已生成 / 失败 / 部分生成 / 编排中）、小节数、字数；顶部有进度条与阶段说明。
- **右侧正文阅读 / 编辑面板**：显示当前选中小节的正文，支持查看、编辑、预览。

#### 3.5.2 生成配置（齿轮按钮）

点击齿轮打开"正文生成配置"对话框，可设置：

| 配置项 | 说明 |
| --- | --- |
| 表格需求 | 不要 / 少量（≤小节数20%）/ 适中（≤40%）/ 大量（默认） |
| 最低字数 | 设为 0 按模型默认长度；大于 0 时低于字数会自动补目录或扩写 |
| 全文一致性审计 | 扩写完成后，先检查并修复与全局事实冲突的内容再配图 |
| 正文生成并发速度 | 同时发起的编排 / 生成 / 扩写 / 审计请求数（不含配图），过高会触发 429 |
| 使用 AI 生图 | 需生图模型状态为"可用"才能开启 |
| 全文图片最大数量 | AI 生图整体决策后择优分布，上限为小节数 |
| 生成 Mermaid 图片 | 适合流程 / 层级 / 时间线 / 关系图，前端渲染，与 AI 生图二选一 |

> Mermaid 转图片当前使用 mermaid.ink 免费接口，可能不稳定，导出 Word 后需核对。

#### 3.5.3 生成流程与阶段

主操作按钮的文字会随状态变化：生成正文 / 暂停 / 继续 / 继续生成正文 / 重新生成正文 / 继续补足字数。后台生成会经历多个阶段，进度条与说明文字会同步切换：

1. **正文编排（planning）**：规划每个小节的写作要点，显示"已完成 X/Y 个小节"。
2. **正文补目录（outline-expanding）**：当设置了最低字数且内容不足时，分轮补充目录层级。
3. **正文扩写（expanding）**：按最低字数目标扩写，显示字数达成百分比。
4. **全文一致性审计（auditing）**：分组检查与全局事实的冲突，发现冲突后自动修复，显示"修复 X/Y 个小节"，无法自动修复的会标注"需人工核对"。
5. **正文配图（illustrating）**：按配置生成 AI 图 / Mermaid 图，显示配图进度。

#### 3.5.4 暂停 / 继续

- 生成中点击"暂停"：会停止调度新的 AI 请求，已发出的请求完成后停止（提示"正在暂停"）。
- 暂停后：可导出当前已完成内容，或点击"继续"恢复。暂停状态下生成配置仅允许修改"并发速度"。

#### 3.5.5 单小节重新生成

- 在左侧目录中，对已生成或失败的叶子小节点击状态标签，弹出"重新生成此小节？"气泡。
- 确认后可填写"本次重新生成的具体要求"（如：强化实施步骤、减少背景描述、突出设备配置），AI 仅覆盖该小节正文。

#### 3.5.6 手动编辑正文

- 选中某个叶子小节，点击"编辑"进入 Markdown 编辑器。
- 支持"预览 / 编辑"切换，编辑完点击"保存"；正文中的图片可点击放大预览。

#### 3.5.7 补足字数对话框

- 若全文已生成、但新设的最低字数未达成，点击生成会弹出选择：
  - **继续补齐字数**：保留已生成正文，仅执行补目录和扩写。
  - **清空重新生成**：覆盖当前全部正文从头生成。

#### 3.5.8 重新生成全文

- 当全部小节均已生成时，主按钮变为"重新生成正文"；执行会先清空全文正文、章节状态和任务进度，再从头生成（有确认提示）。

#### 3.5.9 开发者模式统计

- 在「设置」开启开发者模式后，本页会显示"配图统计"面板：AI 生图与 Mermaid 的计划 / 尝试 / 成功 / 失败 / 跳过数量。

### 3.6 STEP 06 扩写改写（开发中）

- 当前显示"正在开发中，敬请期待"遮罩。规划用于接入旧方案导入、章节扩写和人工校准，暂不可用。

### 3.7 浮动工具条与辅助功能

技术方案各步骤共享一个浮动工具条，分为两组：

**第一组（功能入口）**：
- **重置（危险）**：清空整个技术方案编写进度，有二次确认。
- **首页**：回到 STEP 01 上传招标文件。
- **版本**：打开版本管理对话框。
- **竞品**：打开竞品分析对话框。
- **合规**：打开合规性检查对话框。
- **协同**：打开协同编辑对话框。

**第二组（导航）**：上一步 / 下一步；在"生成正文"步骤会替换为：上一步 / 导出 Word / 继续扩写。

#### 3.7.1 版本管理

- **作用**：保存当前工作进度为一个历史版本，支持查看、对比、恢复历史版本。
- **用途**：在大幅修改目录或正文前先存档，方便回退。

#### 3.7.2 竞品分析

- **作用**：基于评分要求和行业知识，分析竞争策略和关键得分领域。
- **输入**：依赖已完成的「评分分析」结果，以及招标解析得到的项目信息和行业代码。
- **效果**：给出竞争态势判断、关键得分点建议，辅助制定差异化策略。

#### 3.7.3 合规性检查

- **作用**：检查投标文件是否符合招投标法规要求。
- **检查项**：格式合规、资质要求、时间节点、保证金等。
- **输入**：招标解析结果 + 当前技术方案状态。

#### 3.7.4 协同编辑

- **作用**：多人实时协同编辑技术方案，支持多人同时编辑、实时同步、光标显示。
- **操作**：创建协同会话后，会生成会话 ID 供他人加入。

### 3.8 导出 Word

在"生成正文"步骤，浮动工具条提供「导出 Word」。

- **触发条件**：需已生成目录；正文生成 / 暂停处理中时按钮禁用（需先完成暂停）。可导出空目录文档，但建议先生成正文。
- **导出过程**：
  1. 弹出导出进度对话框，显示进度条与说明。
  2. 若正文中包含 Mermaid 图，会通过 mermaid.ink 转换为 Word 图片，速度受网络影响，对话框会提示图片数量。
  3. 完成后提示成功，并列出需要核对的警告（如图片转换提示）。
- **效果**：生成包含正文、表格、图片的完整 Word 文档。导出后建议打开文档核对图片、表格和版式。
- **可取消**：导出过程可取消，取消后提示"已取消导出"。

## 4. 评分分析功能

- **入口**：在 STEP 02 招标文件解析页面触发（启动评分分析任务）。
- **作用**：从招标文件的评分办法中抽取评分项、分值、评分标准，形成结构化评分表。
- **用途**：
  - 供 STEP 03 目录生成的"按评分项对齐"模式使用，让章节覆盖每个评分点。
  - 作为竞品分析的输入，识别关键得分领域。
- **效果**：以后台任务运行，完成后结果保存到技术方案状态（scoringAnalysis），可在相关功能中复用。

## 5. 知识库模块

知识库用于管理"行业 / 通用参考文档"，供目录生成和正文生成参考，提升专业度与模板贴合度。

### 5.1 核心功能

- **文档管理**：上传、查看、删除参考文档；支持按文件夹（folders）分组组织。
- **文档索引**：每个文档有唯一 ID，目录生成步骤可勾选这些 ID 作为参考资料。
- **目录结构**：以文件夹 + 文档（documents）的索引结构（KnowledgeBaseIndex）呈现。

### 5.2 操作流程

1. 进入「知识库」菜单。
2. 创建文件夹分类（如按行业、按文档类型）。
3. 上传参考文档，软件解析后建立索引。
4. 在技术方案 STEP 03 目录生成时，勾选需要参考的知识库文档。

### 5.3 效果

- 被引用的知识库文档会作为模板与专业知识来源，影响目录章节结构和正文写作风格。

## 6. 企业知识库模块

企业知识库面向企业自有资料（资质证书、过往业绩、人员信息、标准模板等），与通用知识库区分，用于沉淀企业专属内容。

### 6.1 核心功能

- **企业资料管理**：分类管理企业资质、业绩案例、组织架构、常用模板等。
- **私有性**：存储企业内部资料，作为标书中资质 / 业绩 / 团队等章节的事实来源。

### 6.2 操作流程

1. 进入「企业知识库」菜单。
2. 按类别录入或上传企业资料。
3. 在编写技术方案或商务标时，引用企业知识库内容填充资质、业绩等部分。

### 6.3 效果

- 与全局事实设定配合，确保标书中涉及企业自身的信息（名称、资质、业绩数据）准确一致。

## 7. 标书查重模块

标书查重用于检测多份投标文件之间的重复度，快速识别围标 / 串标等异常。分为两个步骤：选择标书 → 查重结果。

### 7.1 功能说明

软件首页提示要点：
- 同设备、同用户、同一个 WPS 账号、编辑时间相近等问题，可一秒锁定。
- 可选上传招标文件：多份投标文件都引用了招标文件中的内容时，不计为重复。
- 图片基于哈希校验，只能识别同一张图片；截图、压缩等相似图片无法筛出。

### 7.2 STEP 1 选择标书

- 上传两份或多份投标文件（显示为文件卡片，含文件名、大小、修改时间）。
- 可选上传招标文件作为"基准内容"，避免共同引用招标内容造成误判。
- 可删除已选文件后重新上传。

### 7.3 STEP 2 查重结果（4 个维度）

查重从 4 个维度分别分析，结果以标签页（元数据 / 目录 / 正文 / 图片）切换查看：

| 维度 | 检查内容 |
| --- | --- |
| 元数据 | 设备、账号、编辑时间、作者等隐藏信息 |
| 目录 | 比对章节结构和标题顺序，识别模板化复制 |
| 正文 | 筛查段落、表格和关键描述的重复内容 |
| 图片 | 对原图做哈希校验，定位完全一致的图片 |

### 7.4 效果

- 输出各维度的重复点与重复程度，帮助招标方或投标方识别异常关联，规避围标串标风险。

## 8. 废标项检查模块

废标项检查用于在投标前自查投标文件的风险，避免因废标项响应不当、错别字、逻辑矛盾导致废标或扣分。分为三个步骤：选择标书 → 无效与废标项 → 检查结果。

### 8.1 STEP 1 选择标书（documents）

- **文档角色**：分为"招标文件（tender）"与"投标文件（bid）"两类标签页。
- **文档来源**：
  - **上传解析（upload）**：直接上传文件由软件解析。
  - **技术方案（technical-plan）**：复用当前技术方案模块已生成的内容，无需重复上传。
- **操作**：分别为招标文件和投标文件选择来源并加载内容。

### 8.2 STEP 2 无效与废标项（items）

- **作用**：从招标文件中抽取"无效投标条款"和"废标项"，形成检查清单。
- **解析结果标签页**：
  - **解析结果**：AI 自动抽取的废标项 / 无效项。
  - **自定义检查项**：用户手动补充的检查条目。
- **操作**：核对自动抽取的废标项，按需增删或补充自定义检查项。

### 8.3 STEP 3 检查结果（results）

可选择启用三类检查（默认全开），结果分标签页展示：

| 检查类型 | 说明 |
| --- | --- |
| 废标项检查（rejection） | 根据无效与废标项检查投标文件的响应风险 |
| 错别字检查（typo） | 检查投标文件中的错别字和明显文字错误 |
| 逻辑谬误检查（logic） | 检查前后矛盾、逻辑不一致和表达漏洞 |

- **检查开关**：可在配置中分别开启 / 关闭三类检查（RejectionCheckOptions）。
- **后台任务**：以后台任务运行，显示运行状态。
- **结果呈现**：列出发现项（finding），支持在 Markdown 编辑 / 渲染视图中查看上下文。

### 8.4 效果

- 输出每类风险的具体位置与建议，帮助投标人逐项整改，降低废标概率。
- 错别字与逻辑检查可作为正文交付前的最后一道质量把关。

## 10. 报价管理模块

报价管理是可用的实际功能，用于编制结构化报价单（PricingSheet），自动计算合计金额。

### 10.1 报价单结构

- **项目名称**：报价单所属项目。
- **税率（taxRate）**：默认 13%，可调整。
- **折扣率（discountRate）**：整单折扣比例。
- **分项明细（items）**：每条包含分类、名称、规格、单位、数量、单价、小计、备注。
- **备注（notes）**：报价说明。

### 10.2 分类（CATEGORIES）

报价分项支持 7 类：设备费、人工费、材料费、管理费、利润、税金、其他。

### 10.3 操作流程

1. 填写项目名称、设置税率与折扣率。
2. 点击新增，填写分项（分类 / 名称 / 规格 / 单位 / 数量 / 单价），小计按"数量 × 单价"自动计算。
3. 可编辑或删除已添加的分项。
4. 系统自动汇总：小计合计 → 折扣金额 → 折后金额 → 税金 → 含税总价。
5. 金额按中文千分位格式显示（保留两位小数）。

### 10.4 效果

- 形成清晰的分项报价表，含税前 / 折扣 / 税金 / 含税总价的完整计算，可作为商务标报价附件的基础。

## 13. 常见问题与使用建议

### 13.1 推荐工作流（端到端）

1. **设置** → 配置文本模型（必填）、生图模型（可选）、文件解析器。
2. **企业知识库 / 知识库** → 提前录入企业资质业绩、行业参考模板。
3. **技术方案 STEP 01-02** → 上传招标文件、解析关键信息、启动评分分析。
4. **STEP 03** → 用"按评分项对齐"模式生成目录，勾选知识库参考文档。
5. **STEP 04** → 核对全局事实，确保公司名、项目名、关键数据准确。
6. **STEP 05** → 配置最低字数、表格需求、一致性审计后生成正文，必要时单小节重生成。
7. **导出 Word** → 核对图片、表格、版式。
8. **废标项检查** → 复用技术方案内容做废标 / 错别字 / 逻辑自查。
9. **标书查重** → 多份标书提交前做重复度检测。
10. **报价管理** → 编制分项报价单作为商务标附件。

### 13.2 常见问题

- **Q：AI 功能点了没反应 / 报错？**
  A：先到「设置 → 文本模型」确认已配置并"测试连接"通过。
- **Q：正文生成报 429 错误？**
  A：降低"正文生成并发速度"，该值受 API 提供商限流影响。
- **Q：无法开启 AI 生图开关？**
  A：需到「设置 → 生图模型」配置并测试为"可用"状态。
- **Q：老版 .doc 文件解析失败？**
  A：需本地安装 LibreOffice 后重试。
- **Q：切换页面会中断生成吗？**
  A：不会。所有 AI 任务在后台运行，切换模块不影响，进度会实时回传。
- **Q：Mermaid 图导出 Word 后异常？**
  A：当前用 mermaid.ink 免费接口转换，可能不稳定，导出后需逐张核对。
- **Q：修改目录后正文没了？**
  A：这是预期行为。修改目录 / 全局事实会联动重置后续正文以保证一致性，建议大改前先用「版本」存档。

### 13.3 数据一致性建议

- 编写前在「全局事实设定」中固化公司名称、项目名称、关键参数，并开启"全文一致性审计"，可显著减少各章节前后矛盾。
- 涉及企业自身的资质、业绩数据，统一从「企业知识库」取用，避免手填出错。

### 13.4 风险与质量把关

- 投标前务必走一遍「废标项检查」（废标 / 错别字 / 逻辑三项全开）。
- 多人 / 多公司协作场景，提交前用「标书查重」自查，规避围标串标嫌疑。
