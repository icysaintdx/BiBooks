# 大纲格式渠道深化已完成

## 日期
2026-06-22 21:30

## 责任人
AI 辅助开发

## 背景 / 问题描述
大纲生成的「招标格式优先」路由（tenderStructureWorkflow）在阶段一已实现基础骨架，阶段二已把 bidFileStructure 传给子目录生成。但仍存在三个缺口：

1. **数据结构缺失**：OutlineItem 类型无 `format_ref` 字段，无法存储每个章节应参照的格式模板来源。
2. **解析缺失**：bidFileStructure 中的"按第X章格式编写"这类格式引用信息未被程序化提取，完全依赖 AI 隐式理解。
3. **模板内容未加载**：格式模板实际内容（范本表格正文）在招标文件原文中，但未被单独抽取，大纲生成和后续内容生成无法直接引用。

## 解决方案

### 1. OutlineItem 加 format_ref 字段
`client/src/shared/types/outline.ts` 新增 `format_ref?: string` 可选字段，`normalizeOutlineItem` 透传。纯加法，不影响现有大纲节点。

### 2. 解析 bidFileStructure 提取条目-格式映射
新增 `parseFormatRefsFromBidFileStructure(bidFileStructure)` 函数，正则解析 Markdown 有序列表，提取"条目标题→格式引用"映射（Map）。示例：
```
输入: "1. 投标函 — 按第六章格式1编写\n2. 规划设计方案\n3. 商务标 — 按第六章格式2编写"
输出: Map { "投标函" → "按第六章格式1编写", "商务标" → "按第六章格式2编写" }
```
解析失败返回空 Map（回退现有行为）。

### 3. tenderStructureWorkflow 注入 format_ref
遍历一级条目时，按标题匹配格式引用（含模糊匹配），附加到 `item.format_ref`。子目录生成 prompt（`generateTenderStructureChildrenMessages`）当 `parentItem.format_ref` 存在时追加格式引用提示："招标要求本章按第六章格式1编写，请严格遵循该格式范本展开子目录"。

### 4. 新增格式模板抽取解析项
新增 `formatTemplateTables` 解析项（`output:'markdown'`），从招标文件中提取"投标文件格式""格式范本"等章节的具体格式模板内容（表格、清单、填写说明），按章节分组输出 Markdown。此解析项的结果为后续内容生成阶段"按格式范本填充"提供数据基础。

## 代码示例

### 解析函数
```javascript
function parseFormatRefsFromBidFileStructure(bidFileStructure) {
  const map = new Map();
  if (!bidFileStructure || typeof bidFileStructure !== 'string') return map;
  for (const line of bidFileStructure.split('\n')) {
    const match = line.trim().match(/^\d+\.\s*(.+?)(?:\s*[—–\-]\s*(.+))?$/);
    if (!match) continue;
    const title = match[1].trim();
    const formatRef = match[2] ? match[2].trim() : '';
    if (title && formatRef && !/没有提及|未提及/.test(formatRef)) {
      map.set(title, formatRef);
    }
  }
  return map;
}
```

### 子目录 prompt 增强
```javascript
// 当 format_ref 存在时追加格式引用提示
{ role: 'user', content: `当前固定一级目录：\n编号：${parentItem.id}\n标题：${parentItem.title}\n描述：${parentItem.description || ''}${parentItem.format_ref ? `\n格式要求：招标要求本章${parentItem.format_ref}，请严格遵循该格式范本展开子目录` : ''}` }
```
