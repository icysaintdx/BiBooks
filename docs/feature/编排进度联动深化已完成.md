# 编排进度联动深化已完成

## 日期
2026-06-22 23:30

## 责任人
AI 辅助开发

## 背景 / 问题描述
v0.14.0 已实现编排区基础进度联动：商务标/报价的三态（todo/active/ready）回写编排卡片。但粒度太粗——用户只看到"待处理/进行中/可开始"，不知道具体完成了多少、还缺什么。需要在卡片上显示完成度详情，让用户一眼看出"还缺什么"。

## 解决方案

### 1. OrchestrationBlock 接口新增 detail 字段
`OrchestrationBlock` 接口新增 `detail?: string` 可选字段，`BidOrchestrationBand` 组件在 description 和按钮之间条件渲染 detail。

### 2. 商务标完成度详情
从 `commercialBid.list()` 返回的 `sections[0].form` 中提取：
- 资质：`form.qualifications` 已填名称数 / 总数
- 业绩：`form.projects` 已填名称数 / 总数
- 生成状态：有 result/report 时追加"已生成"
- 输出示例：`资质 3/5 · 业绩 2/3 · 已生成`

### 3. 报价完成度详情
从 `pricing.list()` 返回的 items 和 summary 中提取：
- 报价项：已填名称且单价>0 的项数 / 总项数
- 含税合计：totalAmount > 0 时显示金额
- 输出示例：`报价项 5/8 · 含税 ¥125,000.00`

### 4. 渐进容错
detail 为空字符串时（无数据），不渲染 detail 行为与现状一致。

## 代码示例

### 商务标 detail 计算
```typescript
const form = latestBid.sections?.[0]?.form || {};
const parts: string[] = [];
const quals = Array.isArray(form.qualifications) ? form.qualifications : [];
if (quals.length) {
  const filled = quals.filter((q: any) => q.name?.trim()).length;
  parts.push(`资质 ${filled}/${quals.length}`);
}
```

### 报价 detail 计算
```typescript
const total = Number(latestSheet?.summary?.totalAmount) || 0;
const filledItems = sheetItems.filter((it: any) => it.name?.trim() && Number(it.unitPrice) > 0).length;
const parts: string[] = [`报价项 ${filledItems}/${sheetItems.length}`];
if (total > 0) parts.push(`含税 ¥${total.toLocaleString('zh-CN', { maximumFractionDigits: 2 })}`);
```

### 卡片渲染
```tsx
{block.detail && <p className="bid-orchestration-detail">{block.detail}</p>}
```
