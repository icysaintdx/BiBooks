# 内容生成消费格式范本已完成

## 日期
2026-06-22 22:30

## 责任人
AI 辅助开发

## 背景 / 问题描述
v0.16.0 已落地 `formatTemplateTables` 解析项（从招标文件中提取格式范本表格正文）和 `OutlineItem.format_ref` 字段（存储章节格式来源引用）。但内容生成阶段（`contentGenerationTask.cjs`）尚未消费这些数据——生成正文时 AI 不知道招标要求的格式范本长什么样，也无法根据章节的 `format_ref` 做精细化格式指导。

## 解决方案

### 1. 读取 formatTemplateTables
`runContentGenerationTask` 从 `storedPlan.bidAnalysisTasks.formatTemplateTables` 读取格式范本内容（status guard + trim），传递给编排和内容生成两个 prompt 构建函数。

### 2. 编排 prompt 注入
`buildChapterContentPlanMessages` 新增 `formatTemplateTables` 参数，当内容非空时在 bidAnalysisFactsText 之后注入用户消息："招标文件中的格式范本模板（如本章节有格式要求，请参照对应格式编写正文）"。

### 3. 内容生成 prompt 注入 + format_ref 精细化
`buildChapterContentMessages` 新增 `formatTemplateTables` 参数，当内容非空时在 industryContext 之后注入格式范本模板消息。进一步检查 `parentChapters` 中是否有 `format_ref` 字段，如有则追加精细化提示："本章节所属上级章节的格式要求：{format_ref}。请严格遵循对应的格式范本编写正文。"

### 4. 渐进容错
formatTemplateTables 缺失、为空或解析失败时，prompt 注入逻辑被 `if (String(...).trim())` 保护，不注入任何内容，行为与现状完全一致。format_ref 为可选字段，无则不追加精细化提示。

## 代码示例

### 读取 formatTemplateTables
```javascript
const fttTask = storedPlan.bidAnalysisTasks?.formatTemplateTables;
const formatTemplateTables = fttTask?.status === 'success' ? String(fttTask.content || '').trim() : '';
```

### 内容生成 prompt 注入（含 format_ref 精细化）
```javascript
if (String(formatTemplateTables || '').trim()) {
  messages.push({ role: 'user', content: `招标文件中的格式范本模板（如本章节有格式要求，请严格参照对应格式范本编写正文，保持表格结构、填写说明等与范本一致）：\n${formatTemplateTables}` });
  const parentFormatRef = (parentChapters || []).map((p) => p.format_ref).filter(Boolean);
  if (parentFormatRef.length) {
    messages.push({ role: 'user', content: `本章节所属上级章节的格式要求：${parentFormatRef.join('；')}。请严格遵循对应的格式范本编写正文。` });
  }
}
```
