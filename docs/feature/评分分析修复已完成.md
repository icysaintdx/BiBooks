# 评分分析修复已完成

## 日期
2026-06-22 10:00

## 责任人
开发团队

## 背景 / 问题描述
用户反馈三个问题：
1. Step02 招标文件解析默认为"只解析关键项"，`bidFileStructure`/`bidFileStructureSchema` 无法在默认流程中触发，影响大纲自动生成逻辑
2. "评分分析"模态框标题和描述文字重复出现（`Dialog.Title`/`Dialog.Description` 与 `ScoringAnalysisPage` 内部 header 并存）
3. 点击"开始评分分析"后按钮一直显示"分析中..."，无结果返回

## 解决方案

### Fix 1：默认解析模式改为完整解析
- `TechnicalPlanHome.tsx` 初始 state 的 `bidAnalysisMode` 由 `'key'` 改为 `'full'`
- `requiredBidAnalysisTasks` 由 `getBidAnalysisTasks('key')` 改为 `getBidAnalysisTasks('full')`

### Fix 2：评分分析模态框视觉重叠
- `BidAnalysisPage.tsx` 中 `Dialog.Title` 和 `Dialog.Description` 改为 `className="sr-only"`（无障碍隐藏），视觉上由 `ScoringAnalysisPage` 内部 header 承担

### Fix 3：评分分析事件不触发（核心修复）
根因：`runScoringAnalysisTask` 调用 `updateTask(partial)` 时未传第二参数 `workspaceState`，而 `taskService.cjs` 的 `updateTask` 函数只在 `workspaceState` 为 truthy 时才调用 `emit()`。因此不管进度还是完成，均无 IPC 事件广播给前端。

修复方式（参照 `outlineGenerationTask.cjs` 的两步模式）：
```javascript
// 旧：无 emit
updateTask({ status: 'running', progress: 10, logs });

// 新：先 persist，再带 persistedState 调用 updateTask 触发 emit
let technicalPlan = workspaceStore.updateTechnicalPlan({ scoringAnalysisTask: updateTask({ status: 'running', ... }) });
updateTask({ status: 'running', ... }, technicalPlan);
```

同时新增 `recoverInterruptedScoringAnalysisTask()`，在应用启动 `getActiveTasks()` 时检查并将遗留的 running 状态重置为 error，避免重启后按钮永久禁用。

## 代码示例

```javascript
// scoringAnalysisTask.cjs - 成功路径
const summaryLog = `评分分析完成：...`;
const finalLogs = [...logs, summaryLog];
const finalTask = updateTask({ status: 'success', progress: 100, logs: finalLogs, stats: { ... } });
technicalPlan = workspaceStore.updateTechnicalPlan({ scoringAnalysis: analysis, scoringAnalysisTask: finalTask });
updateTask({ status: 'success', progress: 100, logs: finalLogs, stats: finalTask.stats }, technicalPlan); // 触发 emit
```

```javascript
// taskService.cjs - 启动恢复
function recoverInterruptedScoringAnalysisTask() {
  const technicalPlan = technicalPlanStore.loadTechnicalPlan() || {};
  const scoringAnalysisTask = technicalPlan.scoringAnalysisTask;
  if (!isActiveTaskStatus(scoringAnalysisTask?.status)) return;
  const message = '上次评分分析未完成，请重新分析';
  const recoveredTask = { ...scoringAnalysisTask, status: 'error', ... };
  const state = technicalPlanStore.updateTechnicalPlan({ scoringAnalysisTask: recoveredTask });
  emit(recoveredTask, buildSnapshot(...));
}
```

## 测试结果
- TypeScript `tsc --noEmit` 无错误
- 默认解析模式为"完整解析"，下拉显示"完整解析"
- 评分分析模态框标题不重复显示
- 点击"开始评分分析"后 AI 调用完成，按钮变为"重新分析"，结果正常展示
