# 导出校验深化已完成

## 日期
2026-06-23 00:30

## 责任人
AI 辅助开发

## 背景 / 问题描述
v0.14.0 已实现导出三块一致性硬校验（致命/重大/普通三级拦截），但只检查"有没有"（商务标是否存在、资质数组是否非空），不检查"够不够"（资质是否覆盖招标要求、业绩数量是否达标、报价是否偏离预算）。需要利用 `bidFileStructureSchema` 和 `projectInfo` 中的结构化数据，新增深度校验。

## 解决方案

### 1. buildProjectExportPreview 接收 bidAnalysisTasks
函数新增可选参数 `bidAnalysisTasks`，从中读取 `buildFormSchema`（招标驱动 schema）和 `projectInfo`（含 project_budget）。schema 用于校验资质覆盖和业绩数量，budget 用于报价偏离检查。

### 2. 招标要求资质覆盖校验（major）
从 `formSchema.commercial.qualifications` 中提取 `required=true` 的资质名称，与商务标已填资质做模糊匹配（includes 双向匹配）。缺少的必备资质作为 major 警告提示。

### 3. 业绩数量达标校验（major）
从 `formSchema.commercial.performance.minCount` 读取招标要求的最低业绩数量，与商务标已填业绩数组长度比较。不足时作为 major 警告提示。

### 4. 报价预算偏离校验（minor）
从 `projectInfo.project_budget` 读取招标预算，与报价含税合计比较。偏离超过 30% 时作为 minor 提醒，标注方向（超出/低于）和具体金额。

### 5. 资质关键字段缺失校验（minor）
检查已填资质中是否有未填写证书编号（certificateNo / certificateNumber）的项。缺失数量作为 minor 提醒。

### 6. 渐进容错
所有新校验都受 `latestCommercialBid` 存在性保护。schema 缺失时 `buildFormSchema` 返回默认空 schema，不会产生误报。projectBudget 为 0 时跳过偏离检查。

## 代码示例

### 招标要求资质覆盖
```typescript
const requiredNames = formSchema.commercial.qualifications.filter((q) => q.required).map((q) => q.name);
const filledNames = qualifications.map((q: any) => String(q.name || '').trim()).filter(Boolean);
const missing = requiredNames.filter((name) => !filledNames.some((f: string) => f.includes(name) || name.includes(f)));
if (missing.length) {
  issues.push({ level: 'major', message: `商务标缺少招标要求的必备资质（${missing.join('、')}），请确认是否需要补充。` });
}
```

### 报价预算偏离
```typescript
if (projectBudget > 0 && totalAmount > 0) {
  const deviation = Math.abs(totalAmount - projectBudget) / projectBudget;
  if (deviation > 0.3) {
    const direction = totalAmount > projectBudget ? '超出' : '低于';
    issues.push({ level: 'minor', message: `报价含税合计 ${direction}招标预算超过 30%...` });
  }
}
```
