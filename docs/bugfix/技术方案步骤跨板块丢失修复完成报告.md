BiBooks 桌面端开发

## 背景 / 问题描述

技术方案页采用 8 步线性状态机（`document-analysis → bid-analysis → outline-generation → global-facts → content-edit → expand → delivery-check → export-archive`），当前步数会持久化到 SQLite，跨会话恢复。

实际使用中发现一个状态丢失 Bug：技术方案走到第 6 步（`delivery-check`）或更后（`export-archive`），切换到顶层"商务标""报价"页面再切回技术方案，步数会被重置回第 1 步（`document-analysis`）。

**根因（已定位）**：状态校验数组只列了前 6 个步骤，漏掉了 `delivery-check` 与 `export-archive`，且前后端两处都漏：

1. **前端** `client/src/features/technical-plan/services/technicalPlanStorage.ts` 的 `validSteps`：用于持久化读取时的合法性判断（`isTechnicalPlanState`）。该数组缺 2 步时，状态停留在 `delivery-check`/`export-archive` 的整个 state 被判为非法 → `load()` 返回 `null` → 状态被重置。
2. **后端** `client/electron/services/technicalPlanStore.cjs` 的 `isValidStep`：在 `applyPartial`（写入）、`loadTechnicalPlan`（读取）、内存态重建三处被引用。该函数缺 2 步时，`updateStep('delivery-check')` 在 `applyPartial` 的 `isValidStep(partial.step)` 判断处被拦截 → 步数根本没写进库，回读时自然回落到 `document-analysis`。

两处叠加，导致跨板块切换必然丢步。

## 解决方案

将前后端两处校验集合补齐为与状态机 `steps` 常量完全一致的 8 步列表。仅做对齐式加法改动，不强行抽公共跨进程常量（避免过度抽象）。

1. 前端 `validSteps` 补 `'delivery-check'`、`'export-archive'`；
2. 后端 `isValidStep` 补 `'delivery-check'`、`'export-archive'`。

## 代码示例

前端 `technicalPlanStorage.ts`（补齐后）：

```typescript
const validSteps: TechnicalPlanStep[] = [
  'document-analysis',
  'bid-analysis',
  'outline-generation',
  'global-facts',
  'content-edit',
  'expand',
  'delivery-check',
  'export-archive',
];

function isTechnicalPlanState(state: TechnicalPlanState | null): state is TechnicalPlanState {
  return Boolean(state && validSteps.includes(state.step));
}
```

后端 `technicalPlanStore.cjs`（补齐后）：

```javascript
function isValidStep(value) {
  return ['document-analysis', 'bid-analysis', 'outline-generation', 'global-facts', 'content-edit', 'expand', 'delivery-check', 'export-archive'].includes(value);
}
```

## 测试结果

- **复现路径**：技术方案走到 `delivery-check` → 切到商务标 → 切回技术方案。
  - 修复前：步数重置回 `document-analysis`（第 1 步）。
  - 修复后：步数保持在 `delivery-check`，符合预期。
- **持久化写入**：`updateStep('delivery-check')` 后再读库，`step` 字段为 `delivery-check`（修复前为 `document-analysis`）。
- **回归冒烟**：`client/scripts/project-scoped-modules-smoke.cjs` 通过，商务标 / 报价持久化无回归。
- **类型检查与构建**：`npm run build`（tsc --noEmit + vite build）通过，无类型错误。
