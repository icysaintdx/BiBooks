# 完整标书合成与版式模板方案

更新时间：2026-06-17  
适用范围：BiBooks 桌面版  
目标：把技术方案、商务标、报价、资质附件、业绩附件、版式模板和来源审阅统一合成为最终 Word/PDF。

## 1. 目标

当前系统已有技术方案、商务标、报价管理、修复任务、查重、合规、竞争分析等模块，但它们还分散在各自页面和各自数据层。

本方案的目标是补齐最后一层：

- 统一版式配置。
- 统一附件编排。
- 统一报价注入。
- 统一最终导出。
- 统一导出前校验。

当前状态补充（2026-06-17）：

- 技术方案“导出归档”已经可以在导出时读取当前项目最新商务材料草稿和报价单，拼成“技术标 + 商务标 + 报价文件”的 Word 导出大纲。
- Word 导出服务已支持 Markdown/GFM 表格转 DOCX 真表格。
- 这仍是完整合成器的雏形，不是最终商业版合成器。它还没有正式的版式模板中心、资质/业绩图片自动插入、封面目录模板、PDF 导出、导出前审批和附件完整性校验。

## 2. 三层模型

### 2.1 原始层

输入来源：

- 招标文件解析结果。
- 企业知识库。
- 历史案例。
- 人工录入原文。
- 本地 OCR 识别结果。
- 本地报价脚本结果。

该层只保存原始证据和来源，不直接作为最终文档。

### 2.2 修订层

每次 AI 建议、人工修改或本地脚本修正都形成一个 patch。

patch 要记录：

- 来源模块。
- 原文。
- 建议内容。
- 修改原因。
- 风险级别。
- 来源引用。
- 是否需要人工确认。

### 2.3 发布层

最终导出的 Word/PDF 只取当前生效版本。

发布层不包含：

- 绿色/红色/黄色审阅痕迹。
- 来源提醒。
- 内部审批状态。
- 临时修订建议。

## 3. 统一合成器

建议新增 `finalDocumentComposer`，职责如下：

1. 读取当前项目。
2. 读取技术方案 Markdown。
3. 读取商务标 Markdown。
4. 读取报价数据。
5. 读取资质、业绩、附件图片。
6. 读取版式模板。
7. 合并完整章节顺序。
8. 套用统一样式。
9. 导出 Word。
10. 可选导出 PDF。

当前临时实现：

- 技术方案页内已有 `buildProjectExportOutline`，会读取 `commercialBid.list()` 和 `pricing.list()` 的当前项目最新记录。
- 商务标通过 `buildCommercialBidMarkdown` 转成 Markdown 中间层。
- 报价通过 `buildPricingMarkdown` 转成 Markdown 中间层。
- 然后交给现有 `export.exportWord` 写入 Word。

正式合成器不得长期停留在这个临时实现上，原因如下：

- 商务标 Markdown 里还没有附件图片占位符和企业资料引用。
- 报价 Markdown 只是导出层文本，不是 Excel 报价清单的完整复刻。
- 缺少项目级版式模板选择。
- 缺少导出前总控检查，不会强制拦截未审批来源、缺资质、缺报价、缺附件等问题。
- 缺少 Word/PDF 双输出和导出归档记录。

## 4. 版式模板配置

建议保存为项目级 JSON。当前桌面版设置页已具备多模板配置基础，可以新建、复制、删除和切换模板；后续要把当前项目绑定模板、导出预览和完整合成器继续打通。

字段建议：

- pageSize
- margins
- bindingMargin
- header
- footer
- pageNumberFormat
- fonts
- headingStyles
- paragraphStyles
- tableStyles
- imageStyles
- coverTemplate
- tocTemplate
- appendixStyles

模板需要支持：

- 多行业模板。
- 多项目复用。
- 可导入导出。
- 可视化配置。
- 多配置文件并存。
- 项目级默认模板绑定。
- 字体缺失检测和回退提示。

建议 UI 页面：`版式模板中心`

- 模板列表：工程标、政府采购、信息化项目、运维服务、设备采购等。
- 模板操作：新建、复制、删除、保存、导入、导出。
- 页面设置：A4、页边距、装订线。
- 页眉页脚：Logo、项目名称、投标单位、页码格式。
- 标题样式：一级到四级标题字体、字号、编号、对齐、段前段后。
- 正文样式：字体、字号、行距、首行缩进、段距。
- 表格样式：边框、表头底色、单元格边距、跨页重复表头。
- 图片样式：最大宽度、居中、边框、图注、多图排布。
- 封面/目录：封面字段、目录层级、目录样式。
- 导入/导出模板：保存为本地 JSON，项目绑定模板 ID。
- 字体目录：`client/electron/fonts`，支持放入 `.ttf`、`.otf`、`.woff`、`.woff2`，用于公文字体或客户指定字体。

## 4.1 合并预览编辑层

完整合成器不能直接从各模块数据跳到最终 Word/PDF，中间必须有合并预览编辑层。

预览编辑层职责：

1. 读取技术标、商务标、报价表、附件、版式模板。
2. 生成完整投标文件预览结构。
3. 显示封面、目录、章节、表格、图片和附件位置。
4. 允许人工直接编辑段落。
5. 允许对选中段落调用 AI 改写，但只允许基于当前项目材料、知识库、历史案例或人工输入改写。
6. 允许在指定位置前后插入文字、图片或表格。
7. 所有插入图片必须复制到项目附件目录，并记录来源。
8. 所有表格必须保存为结构化表格数据，导出 Word 时转为真实 Word 表格。
9. 每次改动生成内部修订记录，软件内显示新增、删除、修改，不进入正式导出文档。
10. 预览确认后再进入最终 Word/PDF 导出。

边界：

- 预览编辑层可以显示内部来源、风险、修订、AI 提示。
- 正式导出的 Word/PDF 不显示内部审阅标记。
- 报价金额仍只能由报价模块注入，预览层不允许 AI 编造或替换真实报价。

## 5. 商务标与报价边界

### 商务标

商务标只负责：

- 条款响应。
- 资质摘要。
- 业绩摘要。
- 财务说明。
- 售后承诺。
- 说明性文本。

### 报价

报价只负责：

- 明细项。
- 税率。
- 折扣。
- 汇总。
- 大写金额。
- 控制价/预算对比。

报价不得进入 AI 生成链路。

## 6. 资质和业绩附件

每个资质或业绩条目应包含：

- 标题。
- 编号。
- 有效期。
- 来源文件。
- 图片路径。
- OCR 结果。
- 人工确认状态。

合成规则：

- 先文字摘要。
- 再插入扫描件图片。
- 图片自动缩放。
- 必要时分页。

## 7. OCR 流程

OCR 只作为识别辅助，不作为最终裁决。

流程：

1. 上传扫描件。
2. 本地 OCR 识别。
3. 格式校验。
4. 过期校验。
5. 结果标红/标黄。
6. 人工确认。
7. 通过后入库。

异常规则：

- 格式不对，不可直接保存为已确认。
- 已过期，必须人工处理。
- 识别不清晰，必须人工复核。

## 8. 导出前检查

导出前必须检查：

- 技术方案是否完整。
- 商务标是否缺资质/缺业绩/缺财务说明。
- 报价是否有明细和汇总。
- 版式模板是否绑定。
- 附件是否缺失。
- 来源是否有未审批互联网内容。
- 交付检查是否仍有未处理任务。

导出拦截规则建议：

- 致命问题：未上传招标文件、无正文、报价明细为空、必备资质未确认、存在未审批互联网来源、存在重大合规风险。默认禁止导出正式稿，只允许导出内部检查稿。
- 重大警告：正文过短、附件缺失、报价与商务说明不一致、表格疑似断裂、OCR 识别未确认。允许导出但必须显示确认弹窗并记录内部审批。
- 普通提醒：格式未套模板、图片清晰度风险、来源标注不完整。允许导出，但进入归档摘要。

## 9. 推荐实施顺序

1. 先稳定桌面版现有页面和数据库。
2. 把当前导出归档页升级为“导出前总控”，先把修复任务、来源审批、报价状态、附件状态汇总清楚。
3. 做版式模板中心。
4. 做企业资质/业绩附件库。
5. 做 OCR 识别、格式校验、有效期校验和人工确认锁。
6. 做正式完整标书合成器，替换当前临时 Markdown 拼接大纲。
7. 做 Word/PDF 双输出和归档记录。
8. 最后做平台版协同和权限。

## 10. 模块输入输出关系

| 模块 | 输入 | 输出 | 是否进入正式导出 |
| --- | --- | --- | --- |
| 项目管理 | 招标文件、项目名称、密码、备注 | 项目目录、当前项目上下文 | 不直接进入正文，但决定所有数据边界 |
| 技术方案 | 招标解析、目录、全局事实、知识库 | 技术标正文 Markdown/章节树 | 是，作为技术标章节 |
| 商务标 | 项目信息、企业资料、资质、业绩、财务、售后 | 商务材料 Markdown 中间层 | 是，合成器转成商务标章节 |
| 报价管理 | 报价明细、税率、优惠率 | 本地报价数据、报价表 Markdown 中间层 | 是，最终由本地数据注入 |
| 企业知识库 | 证书、合同、验收报告、图片、OCR 结果 | 可复用企业资产 | 是，作为附件和摘要 |
| 查重 | 当前正文/历史内容 | 重复风险和修复任务 | 不直接进入正式文档 |
| 废标检查 | 招标硬性条款、项目材料 | 废标风险和修复任务 | 不直接进入正式文档 |
| 竞争分析 | 评分规则、竞品信息、行业知识 | 内部策略建议 | 原则上不直接进入正式文档 |
| 合规检查 | 完整项目材料 | 合规问题和修复任务 | 不直接进入正式文档 |
| 交付检查 | 各模块问题、来源审批、附件状态 | 导出前总控结论 | 不进入正文，但决定是否可导出 |
