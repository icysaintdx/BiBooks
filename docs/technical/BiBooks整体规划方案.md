#### 3.1 生成封面模块（1天）

**新增文件**：

**功能**：
- 根据项目名称、投标单位、日期生成标准封面

**参考**：BiaoShu-SKILL 的 `generate_placeholder.py`

一、项目核心可行性判定（核心结论：100% 可落地、高度自动化、商业价值极强）
1.1 行业现状与痛点
目前招投标行业100% 人工低效作业，是传统行业数字化空白赛道，核心痛点如下：
重复性极高：80% 标书内容为固定模板（公司资质、营业执照、体系证书、过往案例、财务报表、售后方案），仅 20% 为项目定制内容（招标需求响应、项目方案、报价明细）；
出错率极高：人工核对资质、工期、资质匹配度、格式排版、错别字，极易导致废标，行业废标率超 15%；
耗时极长：普通标书人工制作 1-3 天，复杂工程标、服务标需 3-7 天，人力成本高昂；
门槛依赖人：依赖资深标书文员，人员流动直接导致业务断层，无标准化流程。
1.2 项目实现可行性
结合脚本自动化 + 大模型 AI双能力，可实现全流程无人化做标，分为两个层级：
初级脚本自动化（零 AI、纯代码）：固定模板填充、数据自动匹配、格式统一、资质自动校验、目录自动生成、PDF 自动排版，100% 可实现；
高级 AI 智能生成：自动解析招标公告、招标文件，智能撰写定制化技术方案、服务承诺、应急预案、人员配置方案，智能响应招标评分要点，95% 场景可全自动生成；
全流程闭环：数据入库→智能匹配模板→AI 撰稿→自动排版→合规校验→导出 Word/PDF→一键自查废标风险，完全替代人工。
1.3 核心优势（差异化市面所有工具）
市面现有标书工具仅为模板库 + 简单填空，无自动化、无 AI 解析、无合规校验；本项目为全自动智能化系统，无需人工改格式、无需人工对参数、无需人工写方案，是行业降本刚需工具。
二、项目整体产品定位
2.1 产品名称
AI 全自动标书智能生成系统
2.2 产品定位
面向建筑工程、物业服务、设备采购、软件开发、运维服务等全行业招投标场景，一站式无人化标书制作平台，实现「上传招标文件 + 导入企业数据→全自动输出合格投标文件」。
2.3 核心功能目标
自动化解析 PDF/Word 招标文件，提取招标要求、评分标准、资质门槛、工期要求、人员要求；
企业资质、案例、财务、人员、证书永久数据库，自动调用、自动匹配；
固定内容脚本自动填充，定制内容 AI 智能撰写；
全自动格式标准化（字体、行距、页边距、目录、页码、页眉页脚）；
AI 智能废标风险检测（资质不符、漏项、格式错误、响应不齐全）；
一键导出规范投标 Word、PDF 文件，支持批量做标。
三、系统整体技术架构（分层架构、可直接开发落地）
整体采用前后端分离 + 脚本自动化 + AI 大模型接口 + 数据库存储架构，轻量化、易部署、可本地 / 云端部署，适配个人、工作室、企业使用。
3.1 架构分层（5 层完整架构）
第一层：用户交互层（前端）
Web 管理后台，支持：文件上传、数据录入、模板管理、标书生成、预览导出、风险检测、日志查看。
第二层：核心业务逻辑层（后端核心）
文件解析模块：解析 PDF/Word 招标文件，结构化提取关键信息；
脚本自动化模块：核心核心！通过代码实现文档批量填充、格式修改、目录生成、图片排版、数据对齐；
AI 智能创作模块：对接大模型，根据招标要求生成定制化技术方案；
合规校验引擎：匹配招标规则 + 企业资质，自动排查废标风险；
模板管理引擎：分行业标书模板动态管理。
第三层：AI 能力层
支持公有大模型 API（低成本）+ 本地私有部署大模型（隐私版） 双模式：
通用场景：对接 Deepseek、智谱、通义千问 API，性价比高、生成质量高；
企业私密场景：本地部署开源大模型，标书数据不外泄。
第四层：数据存储层
企业私有数据库，存储所有固定资料：营业执照、资质证书、人员证件、过往中标案例、财务报表、报价模板。
第五层：输出渲染层
自动渲染标准投标 Word 文档、高清 PDF，适配所有招投标平台格式要求。
四、精准技术栈（适配自动化 + AI 开发，轻量化高效）
结合自动化脚本、AI 开发、web 部署最优技术组合，兼顾开发速度、运行稳定性、自动化能力
4.1 后端核心技术栈（核心自动化能力）
Python（核心主力）：所有自动化脚本、文件处理、AI 调用、数据处理全部基于 Python，生态最完善；
文档自动化库：
python-docx：Word 文档全自动编辑、填充、改格式、插目录、改页眉页脚；
pdfplumber/PyPDF2：招标文件 PDF 解析、文本提取、规则识别；
reportlab：PDF 自动生成与排版；

标书报价隐私风控完整落地方案：解决「报价数据必须入 AI 校验 + 不能外传 + 不用全量本地大模型部署」矛盾

一、先拆分两类数据：严格物理隔离（最基础红线）
把标书数据拆成 **【涉密私有域数据】和【可出域非涉密文本】**，从源头分开流转，任何场景不混传第三方 API：
1. 涉密私有域（严禁明文上传任何公有大模型：DeepSeek / 通义 / 智谱等云端 API）
报价单价、分项报价、税费、总价、成本价、分项工程量、内部成本测算、合作底价、独家商务条款、涉密人员薪酬、涉密项目合同金额、内部优惠方案
所有报价表 Excel、报价明细表格全归入本地数据库，全程留存在用户本地电脑 / 企业私有服务器，永不明文出境。
2. 非涉密可出域（可丢给云端公有 AI）
招标需求原文、技术方案描述、售后服务内容、公司通用资质、项目建设方案、施工工艺、管理制度、通用承诺条款（无价格）、评分细则（招标公告公开内容）
关键拆分规则：
报价核算、价格勾稽、报价差错校验，全部交由【本地代码脚本 + 本地微型小模型】处理，不调用任何云端大模型；
只有技术文案、方案撰写部分按需调用云端公有 AI，这部分内容不带任何数字报价。
解决你第一个矛盾：AI 要审报价≠报价必须发给云端大模型。价格校验不用通用大模型，靠 Python 脚本 + 轻量化本地小模型做数理验算。
二、四层技术落地方案（分 3 种付费档位，用户按需选择，不用全量部署百亿参数大模型）
方案 1：标准版（90% 中小投标公司首选，零大模型部署成本、完全杜绝报价外泄，商用主力版本）
核心逻辑：报价全本地脚本化计算校验，纯文本方案走云端 AI，数据物理割裂
报价计算 & 校验模块：纯 Python 脚本实现（无任何 AI 参与，完全本地）
用 Pandas+openpyxl 开发固定报价验算引擎：
分项单价 × 工程量 = 合价、分项汇总→小计→含税总价、税率自动核算、大小写金额自动换算、报价前后数据一致性校验（标书正文报价和报价表总价不一致自动标红预警）；
所有报价规则写成配置化规则引擎（配置税率、规费、取费标准），纯代码做数学验算，不需要任何大模型读取价格。
废标报价校验：招标要求限价、最高控制价（公开信息可单独上传），脚本自动比对投标总价是否超限价、漏项缺报价，全部本地运算。
通俗说：算账、审报价对错这件事，用代码就能搞定，本来就不需要 AI，之前 AI 只是锦上添花，不是刚需。
标书技术文案：剥离全部价格字段后再传给云端 DeepSeek 等 API
系统自动过滤全文所有金额、单价、总价数字，把脱敏后的纯文字内容发给云端 AI 生成方案；
AI 返回的技术方案文本落地本地后，系统再自动从本地私有报价库回填对应无涉密的通用数据，云端 AI 自始至终看不到任何报价。
脱敏规则自动化：
系统内置正则 + 表格字段识别，自动识别表格中「单价、合价、总价、费率」列，替换为占位符{报价字段_1}，云端 AI 只看到占位符，看不到真实金额。
适用：绝大多数小微企业、标书工作室，零本地模型部署成本，云端 AI 看不到报价，报价全锁本地，解决泄密痛点。

用户担心连非涉密文本也不想外流，采用：小参数量本地轻量模型处理文案，大额通用云端 AI 完全停用

所有文案生成 + 报价校验全在本地完成，完全断网使用，不接入任何第三方云端 API；
报价数据、企业资料全在本地，数据 100% 不出内网，无任何外泄通道；
硬件成本：普通办公电脑即可，无需专业服务器，对比私有化百亿大模型（几十万成本）成本下降 99%，是中型企业最优解。
方案 3：极致涉密版（国企 / 涉密项目 / 军工投标，内网离线全闭环）
内网物理断网，本地部署微调后的行业垂直小模型，全程不联网、无外网 API 调用，全流程离线自动生成标书，报价、资质、商务数据全程内网闭环。
三、补充兜底：如果极端场景需要 AI 少量参考报价逻辑（极少场景）的脱敏处理
个别场景需要 AI 参考报价逻辑，但不想明文外传，采用哈希脱敏 + 数值偏移算法：
真实报价在本地做随机数值偏移、哈希不可逆加密，云端只能拿到无实际业务意义的变形数字，无法反推真实底价；
AI 仅分析报价结构逻辑，无法还原真实报价成本，原始明文报价永久留存在本地数据库。

SaaS 云端版（中小客户）：采用方案 1，报价全本地留存、文案去价后上云 AI，按年订阅，客户不用部署任何程序，软件安装在客户本地电脑，报价数据库存在客户本机，服务商后台无法读取用户报价（产品关键合同条款写明：服务商无权抓取本地报价库）；

六、纠正一个关键误区：报价校验不需要大模型
很多人误区：审核报价、核对汇总、查价格错误必须大模型 AI，实际：
数学汇总、税率核算、表格勾稽属于确定性规则计算，是传统 Excel / 代码强项，规则固定，用 Python 规则引擎 100% 精准实现，出错率远低于 AI；
