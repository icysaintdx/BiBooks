## 架构决策记录

### 2026-06-05: 部署架构选择
- **决策**: 采用方案 C（桌面版为主 + 轻量同步层）
- **原因**: 保留桌面版离线能力 + 支持团队协作
- **实施路径**: Phase 1 单机 → Phase 2 加 HTTP API → Phase 3 切 PostgreSQL → Phase 4 协作功能

### 2026-06-05: 基础项目选择
- **决策**: 使用 OpenBidKit_Yibiao 作为二开基础
- **原因**: 完整可用代码，功能覆盖面广，比 BiaoShu-SKILL（无代码）更高效
- **BiaoShu-SKILL**: 作为流水线参考标准，移植行业检测和 AI 去痕模块
